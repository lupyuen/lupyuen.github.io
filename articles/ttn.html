                        Take Note: This is missing from the RAKwireless docs‚Ä¶

                        3. Open the Downloaded global_conf.json with a text editor.

                        4. On our RAKwireless Gateway, run this‚Ä¶

                        5. Select ‚ÄúEdit Packet Forwarder Config‚Äù

                        6. Look for the gateway_conf section‚Ä¶

                        7. Replace these values from the Downloaded global_conf.json‚Ä¶

                        4.1 Join Commands

                        Head over to the Serial Terminal for PineDio Stack.

                        At the PineDio Stack Command Prompt, enter these commands to join PineDio Stack to The Things Network‚Ä¶

                        1. First we start the Background Task that will handle LoRa packets‚Ä¶

                          create_task

                        2. Next we initialise the LoRa SX1262 and LoRaWAN Drivers‚Ä¶

                          init_lorawan

                        3. Set the DevEUI‚Ä¶

                          las_wr_dev_eui 0xAB:0xBA:0xDA:0xBA:0xAB:0xBA:0xDA:0xBA

                          Change ‚Äú0xAB:0xBA:...‚Äù to your DevEUI

                          (Remember to change the ‚Äú,‚Äù delimiter to ‚Äú:‚Äù)

                        4. Set the JoinEUI‚Ä¶

                          las_wr_app_eui 0x00:0x00:0x00:0x00:0x00:0x00:0x00:0x00

                          Change ‚Äú0x00:0x00:...‚Äù to your JoinEUI

                          (Yep change the ‚Äú,‚Äù delimiter to ‚Äú:‚Äù)

                        5. Set the AppKey‚Ä¶

                          las_wr_app_key 0xAB:0xBA:0xDA:0xBA:0xAB:0xBA:0xDA:0xBA0xAB:0xBA:0xDA:0xBA:0xAB:0xBA:0xDA:0xBA

                          Change ‚Äú0xAB:0xBA:...‚Äù to your AppKey

                          (Again change ‚Äú,‚Äù to ‚Äú:‚Äù)

                        6. Finally we send a request to join The Things Network‚Ä¶

                          las_join 1

                          (‚Äú1‚Äù means try only once)

                        5 Send Data to The Things Network

                        Finally we‚Äôre ready to send data from PineDio Stack to The Things Network!

                        We should see 5 bytes of 0x00 received by The Things Network‚Ä¶

                        7 Fair Use of The Things Network

                        The Things Network is Free for Fair Use‚Ä¶

                        How many messages can we send in an hour?

                        Each device may transmit roughly 10 tiny messages per hour.

                        (Assuming 12 bytes per message)

                        This varies by region, message size and data rate, as explained here‚Ä¶

                        • ‚ÄúFair Use Policy Explained‚Äù

                        TLDR: We can send more messages to the network if we‚Ä¶

                        1. Reduce the Message Size

                          (Payload should be 12 bytes or smaller)

                        2. Select a Higher Data Rate

                          (Our LoRaWAN Driver uses DR2, which is 125 kbps)

                        Why does the message rate vary by region?

                        The Things Network operates on ISM Radio Bands, which are regulated differently across regions.

                        To comply with Local Regulations, each device is allowed to transmit data for up to X seconds per day. (Where X depends on the region)

                        This daily limit is known as the Duty Cycle, as explained here‚Ä¶

                        • ‚ÄúThe Things Network: Duty Cycle‚Äù

                        How can we optimise our messages?

                        Encode our message payload with CBOR (Concise Binary Object Representation) instead of JSON.

                        (CBOR works like a compressed, binary version of JSON)

                        This JSON Payload occupies 10 bytes‚Ä¶

                        { "t": 1745 }

                        While the CBOR version needs only 6 bytes!

                        (More about CBOR)
